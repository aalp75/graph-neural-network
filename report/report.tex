\section{Introduction}

This repository aims to implement the paper \textit{Neural Execution of Graph Algorithms}~\cite{velickovic2019neural} and to reproduce the results. The model is focus to predict to next state in the graph algorithm iteration. In addition to the original tasks, it includes an extension to the connected components algorithm.

This report provides a concise overview of the implementation, experimental setup, and results, and serves as supporting notes to better understand both the repository and the model.

\section{Method}

\subsection{Graph Generation}

Following the original paper, the training data is generated using seven types of undirected random graphs:
\begin{itemize}
\item Ladder graphs
\item 2D grid graphs
\item Trees (generated using Pr\"ufer sequences)
\item Erd\H{o}s--R\'enyi graphs
\item Barab\'asi--Albert graphs
\item 4-community graphs
\item 4-caveman graphs
\end{itemize}

\subsection{Model Architecture}

The repository implements the maximisation-based Message Passing Neural Network (MPNN)~\cite{gilmer2017neural}, which is the best-performing model reported in the original paper.

The architecture consists of:
\begin{itemize}
\item an \textbf{encoder} $f_A$ and a \textbf{decoder} $g_A$, both defined per algorithm,
\item a shared \textbf{processor} $P$,
\item a \textbf{predecessor network} $R$ for tasks such as Bellman-Ford and Prim,
\item a \textbf{termination network} $T_A$ predicting when the algorithm should stop.
\end{itemize}

Compare to the original paper the termination network has been slighly modified. Additional details can be found in~\cite{velickovic2022clrs} and the computation pipeline can be described as follows:

\[
\begin{aligned}
&
&&
\;\xrightarrow{\makebox[1.5cm]{\text{Decoder}}}\; y^t = g_A(h^t, z^t)
\\[8pt]
(x^{t}, h^{t-1}, G)
&\;\xrightarrow{\makebox[1.5cm]{\text{Encoder}}}\;
z^{t} = f_A(x^{t}, h^{t-1}) \;\xrightarrow{\makebox[1.5cm]{\text{Processor}}}\;
h^{(t+1)} = P(z^t, G)
&&
\;\xrightarrow{\makebox[1.5cm]{\text{Predecessor}}}\; \pi^t=R(h^t, G)
\\[8pt]
&&&
\;\xrightarrow{\makebox[1.5cm]{\text{Termination}}}\; \tau^{(t)} = T_A(h^t, \bar{h}^t, y^t)
\end{aligned}
\]

Let $N$ be the number of nodes and $K$ the latent dimension. The variables have the following shapes:
\[
x \in \mathbb{R}^{N}, \quad
z \in \mathbb{R}^{N \times K}, \quad
h \in \mathbb{R}^{N \times K}, \quad
y \in \mathbb{R}^{N}, \quad
\pi \in \mathbb{R}^{N \times N}, \quad
\tau \in \mathbb{R}.
\]

The loss function depends on the algorithm. Prim's algorithm is trained to predict the next node in the MST.
\begin{itemize}
\item Mean Squared Error (MSE): Bellman-Ford distances, Connected Components
\item Binary Cross-Entropy (BCE): BFS reachability and termination prediction
\item Categorical Cross-Entropy (CE): Prim's next node selection and predecessor prediction
\end{itemize}

Experimental setup:
\begin{itemize}
    \item Latent dimension $K = 32$
    \item Optimizer Adam
    \item Learning rate $5 \times 10^{-4}$
\end{itemize}

The implementation is done using PyTorch (2.10.0)

\subsection{Train}

The model is trained on a dataset composed of 100 graphs per category (700 in total). Early stopping is applied based on a validation set (20 graphs per category, 140 in total) with a patience of 10 epochs. We add a minimum number of epochs because he training phase is very unstable at start due to the termination process which seems slow to learn. Training takes approximately one hour on a CPU/GPU setup (Intel Core i7-12700K/RTX 4060 Ti)

\subsection{Test}

The model is tested on a set of 20 random graphs of varying numbers of nodes. Increasing the number of nodes allows us to assess how well the model has learned the algorithms, i.e., whether it is able to generalise the algorithmic reasoning.

\section{Extension}

\subsection{Details}

The framework requires algorithms to be Markovian processes, i.e., the next state depends only on the current state. Algorithms such as DFS are non-Markovian and cannot be trained in this framework. Dijkstra's algorithm can be reformulated in a Markovian way by augmenting the node features with a visited state, but this would significantly increase the complexity of the code (It is actually also the case for DFS algorithm)

A natural extension of the framework is connected components (CC), which can be formulated as a Markovian process as follows:

\[
\text{CC : }x_i^{(1)} = i \hspace{3cm} \text{CC : }x_i^{(t+1)} = \min_{j \in \mathcal{N}_i} (x_i^{(t)}, x_j^{(t)})
\]

\subsection{Results}

Connected components is an algorithm closely related to BFS, so strong results are expected. It can be interpreted as a multi-source BFS. Unlike the original paper, which primarily focuses on connected graphs, it has been tested on sparser graphs with multiple components to fully assess the performance of the algorithm.
